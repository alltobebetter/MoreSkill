# 快速入门

欢迎来到 MoreSkill —— 一个面向所有开发者的开源技术知识库。

## 这个项目是什么

MoreSkill 不是教程，也不是面试题集。它是一份**技术地图**，用最直白的语言帮你理解每项技术"是什么"和"为什么要用它"。

我们相信：知道一个技术解决什么问题，比知道它怎么用更重要。

## 为什么做这个

AI 时代发展很快，工具越来越强大，但如果你不了解背后的技术，AI 对你来说就是一个黑盒。

比如 AI 告诉你"这个项目应该用 Redis 做缓存"，如果你连 Redis 是什么都不知道，你会心慌，你不敢做决定。但如果你花几分钟了解了"哦，Redis 就是一个基于内存的键值存储，读写特别快，适合做缓存"，你就能理解这个建议是合理的。

**MoreSkill 就是帮你建立这种认知的地方。**

## 如何使用

1. 从左侧目录选择你感兴趣的主题
2. 每篇文档都从"是什么"和"为什么"讲起，配合比喻和例子
3. 重点理解设计思想，不需要死记 API
4. 右上角可以复制 Markdown、导出 PDF，或直接跳转 GitHub 编辑

## 一起来写

这个项目目前可能很小众，但如果这些内容帮助到了你，哪怕只是一点点，我们非常欢迎你一起参与。

**你不需要是专家。** 你可以：

- 修正一个错别字或不准确的描述
- 用更好的比喻重新解释一个概念
- 补充一个你觉得缺少的技术话题
- 翻译内容为其他语言

### 如何参与

1. Fork 这个仓库：[github.com/alltobebetter/MoreSkill](https://github.com/alltobebetter/MoreSkill)
2. 文档都在 `docs/` 目录下，每篇是一个 `.md` 文件
3. 目录结构在 `docs/content.md` 中定义
4. 写完后提交 Pull Request，我们会尽快 Review

你也可以直接点击每篇文档右上角的「编辑」按钮，会跳转到 GitHub 对应文件的编辑页面。

### 写作风格

- 用"是什么"和"为什么"开头，而不是直接上代码
- 多用比喻和生活中的例子，让没接触过的人也能理解
- 不需要写得很深，重点是讲清楚这个技术解决了什么问题
- 保持轻松友好的语气，像是在和朋友聊天

> 每个人都曾经是初学者。如果你曾经因为一篇好文章而豁然开朗，现在你也可以成为写出那篇文章的人。
